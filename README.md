## 1. რეპოზიტორიის ატვირთვა GitHub-ზე

```bash
cd real-estate-monitor
git init
git add .
git commit -m "Initial commit"
```

შექმენით ახალი რეპოზიტორია GitHub-ზე (public-ისთვის GitHub Pages
ავტომატურად და უფასოდ მუშაობს; private რეპოზიტორიისთვისაც შესაძლებელია,
თუ თქვენი ანგარიში მხარს უჭერს Pages-ს) და გააგზავნეთ:

```bash
git remote add origin https://github.com/<თქვენი-username>/real-estate-monitor.git
git branch -M main
git push -u origin main
```

## 2. GitHub Pages-ის ჩართვა

რეპოზიტორიის გვერდზე: **Settings → Pages**
- Source: **Deploy from a branch**
- Branch: **main**, ფოლდერი: **/docs**
- შეინახეთ (Save)

რამდენიმე წუთში გვერდი ხელმისაწვდომი იქნება მისამართზე:
`https://<თქვენი-username>.github.io/real-estate-monitor/`

ეს ბმული შეინახეთ ტელეფონში (სანიშნედ ან მთავარ ეკრანზე).

## 3. ტესტირება

**Actions** ტაბში აირჩიეთ „Real Estate Monitor" workflow და დააჭირეთ
**Run workflow** (ხელით გაშვება) — რამდენიმე წუთში `docs/index.html`
განახლდება და შეგიძლიათ ნახოთ გვერდზე. თუ რამე შეცდომაა, დეტალები ჩანს
Actions-ის ლოგებში.

**საიდუმლოებები (secrets) არ სჭირდება** — workflow-ს იყენებს GitHub-ის
ჩაშენებულ `GITHUB_TOKEN`-ს გვერდის ავტომატურად commit/push-ისთვის.

## შეზღუდვები და შესაძლო გაუმჯობესებები

- **ss.ge**-ს `createDate` ნამდვილად ნიშნავს განცხადების შექმნის თარიღს.
  **myhome.ge**-ს (`last_updated`) და **korter.ge**-ს (`actualizeTime`)
  ველები კი ფაქტობრივად „ბოლო განახლების/დადასტურების დროს" აღნიშნავს —
  ეს ნიშნავს, რომ ზოგჯერ ძველი, უბრალოდ "აწეული"/განახლებული
  განცხადებაც შეიძლება გამოჩნდეს, როგორც "ახალი". ეს ამ საიტების
  მონაცემების შეზღუდვაა, არა კოდის ხარვეზი.
- **ss.ge** და **korter.ge**-ს არ აქვთ საიმედო "უახლესით დახარისხების"
  URL პარამეტრი, ამიტომ სკრიპტი ასქროლავს რამდენიმე გვერდს
  (`MAX_PAGES` თითოეულ სქრეპერში) და თარიღით ფილტრავს. თუ შეამჩნევთ,
  რომ ახალი განცხადებები გამოტოვებულია, გაზარდეთ `MAX_PAGES`
  შესაბამის ფაილში.
- **korter.ge**-ს ლარში (GEL) მითითებული ფასები კონვერტირდება USD-ში
  მიახლოებითი კურსით (`GEL_TO_USD_RATE` — `scraper/korter_ge.py`).
- ამჟამად მხოლოდ თბილისია მოცული. სხვა ქალაქის დასამატებლად საჭიროა
  შესაბამისი `cityIdList`/URL-ის მოძებნა თითოეულ საიტზე.
- პირველი/ხელით გაშვება ("manual") იყენებს "ბოლო 7 საათის" ზღვარს,
